HTML

  • HyperText Markup Language: foundation to a webpage, think of foundation to a house
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML makes websites readable and easy to use; adds structure to the content of a webpage

CSS

  • Cascading Style Sheets: language used by developers to define how webpage content is presented to users
  • CSS defines the colors, aesthetics, visual layout, and organization
  • Inline CSS - style an element directly in the HTML file; useful for quick and permanent changes
  • Example of inline css is using style attribute to change the color of a single heading
  • internal style sheet - contains CSS rules for the webpage in the head section of the HTML file
  • External CSS Style Sheet - most common way to style a webpage; preferred in most cases because it allows developers to keep all their CSS rules in a separate file (use of "link" element)
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches to main branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: pulls down the latest main branch
  • git commit -m "description"

JavaScript

  • JavaScript - adds interactivity to website
  • High-level dynamic programming language that allows users to interact with websites
  • Critial tool in every web developers tool kit
